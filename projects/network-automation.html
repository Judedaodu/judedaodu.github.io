Automation & Monitoring · Project 02

Network Infrastructure Automation & Real-Time Monitoring Portal

Network Automation with AWS EC2, Python, Flask, & Hybrid Hardware Environments

Project Overview

Designed and implemented an automated network configuration and monitoring solution for a rapidly growing fintech platform to streamline infrastructure management and ensure enterprise-grade disaster recovery. The project resolved critical operational bottlenecks across a multi-vendor network environment by integrating a custom-built automation engine with an interactive, centralized dashboard deployed on Amazon EC2. The architecture combines Python and shell scripting for scheduled, reliable configuration backups, a lightweight Flask web framework for real-time health telemetry, and secure networking protocols to communicate across production systems. This centralized platform minimizes configuration drift, ensures immediate recovery readiness, and delivers end-to-end visibility across all active corporate networking infrastructure.

Problems & Goals

Problems

  • ✗ The existing network infrastructure relied heavily on manual backups, consuming significant engineering hours and increasing the risk of human error or missed schedules.
  • ✗ Managing a hybrid environment consisting of both Cisco and Huawei devices required switching between disparate operating systems and command syntaxes, leading to operational fragmentation.
  • ✗ Network health and configuration status lacked real-time monitoring and centralized visibility, delaying the identification of silent device failures or unauthorized changes.
  • ✗ The absence of an automated, off-site repository for network states increased the recovery time objective (RTO) and exposed the business to potential extended downtime during outages.

Goals

  • ✓ Build a scalable, vendor-agnostic network automation engine capable of orchestrating configurations across diverse Cisco and Huawei enterprise devices seamlessly.
  • ✓ Deliver a modern, centralized web dashboard providing real-time infrastructure telemetry, backup validation logs, and current network health status.
  • ✓ Transition the automation tools from localized scripts to a secure, cloud-native hosting model on Amazon Web Services (AWS) for maximum uptime and resilience.
  • ✓ Eliminate operational bottlenecks by targeting a near-total reduction in manual management overhead for standard network administration tasks.

Role & Responsibilities

  • ▸ Architected the core automation framework, including the scheduled backup engine, multi-vendor syntax parsing logic, and the central data aggregation pipelines.
  • ▸ Developed modular Python automation scripts utilizing industry-standard libraries to interface with Cisco IOS and Huawei VRP systems securely.
  • ▸ Built and deployed the web-based monitoring dashboard using the Flask framework, incorporating clean user interfaces for real-time telemetry display.
  • ▸ Provisioned, secured, and managed the cloud infrastructure on AWS EC2, ensuring secure network paths via VPC configurations, security groups, and IAM policies.
  • ▸ Implemented comprehensive error handling, logging, and state validation mechanisms within the backup engine to verify file integrity upon every execution.
  • ▸ Collaborated with network operations teams to optimize script execution schedules, device credential management, and data retention rules.

Architecture & Deployment Flow

  • → Network infrastructure devices (Cisco routers/switches running IOS and Huawei hardware running VRP) operate within secure network segments.
  • → A centralized automation platform is deployed within a secure Virtual Private Cloud (VPC) on an Amazon EC2 instance.
  • → Chronologically scheduled tasks (cron jobs) on the EC2 instance trigger the Python and shell scripting core backup engine.
  • → The automation engine establishes secure SSH/SFTP sessions to target devices, executes configuration retrieval commands, and parses vendor-specific command-line outputs.
  • → Retrieved configuration states are parsed, timestamped, and safely written to a secure, organized file repository on the cloud instance for disaster recovery.
  • → Simultaneously, the scripting engine captures device availability metrics, connection latency, and execution success statuses, outputting structured data logs.
  • → A Flask web application running on the EC2 instance reads these real-time logs and state repositories to dynamically render a monitoring UI.
  • → Network administrators access the interactive dashboard via web browsers, where traffic is securely routed through defined AWS security groups.
  • → The dashboard UI visualizes individual device statuses, tracks historical backup trends, flags active connection issues, and provides direct downloads for historical configuration files.

Key Implementation Details

Multi-Vendor Python Automation Engine

Developed highly adaptable scripts utilizing specialized network automation libraries to normalize interactions across different network operating systems. The software intelligently detects device types, maps corresponding command syntaxes, cleans raw CLI outputs, and validates that configurations are completely downloaded before marking a job successful.

Flask Telemetry Dashboard

Built a lightweight, performant frontend-backend system using Flask to eliminate command-line dependencies for status tracking. The application consumes structured text and log data generated by the backend scripts, providing non-blocking UI updates on device availability, historical backup integrity, and current system alerts.

Cloud Deployment & Infrastructure Hardening

Migrated the entire solution from a fragmented, local execution model to a dedicated AWS EC2 environment. Implemented stringent infrastructure security controls, restricting ingress and egress traffic using custom security groups, anchoring the compute within a private subnet layout, and leveraging IAM roles for secure credential and system isolation.

Results & Impact

  • ✓ Successfully eliminated manual operational overhead by 99%, converting hours of routine maintenance into hands-free, automated background operations.
  • ✓ Achieved 100% backup compliance and reliability across all production Cisco and Huawei devices, creating a dependable, auditable disaster recovery repository.
  • ✓ Accelerated incident discovery and response times from hours to real-time via visual alerts on the centralized monitoring dashboard.
  • ✓ Established a standardized, modern automation blueprint within the organization that can scale easily to accommodate additional hardware vendors or new sites.
