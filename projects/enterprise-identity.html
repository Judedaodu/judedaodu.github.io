Enterprise IT · Project 06

Enterprise Identity Infrastructure & Hybrid Systems Administration

Identity Access Management (IAM), Automated Patching, and Cross-Platform Service Reliability

Project Overview

Designed, managed, and optimized a secure, high-availability hybrid identity and systems infrastructure for an enterprise environment serving over 5,000 users. The project consolidated disparate identity vectors into a unified, secure access model using Active Directory (AD) and Microsoft 365 (M365) while systematically hardening the enterprise attack surface. Concurrently, orchestrated automated OS patching, configuration baseline enforcement, and critical vulnerability remediation pipelines across virtualized VMware environments hosting core Linux and Windows service clusters. By establishing proactive systems administration workflows and strict Service Level Agreement (SLA) compliance tracking, the platform sustained maximum uptime for foundational enterprise systems, safeguarding business continuity and corporate digital assets.

Problems & Goals

Problems

  • ✗ Fragmented Identity & Loose Access Controls: Operating without centralized identity lifecycle controls led to credential sprawl, orphaned user accounts, and unauthorized access permissions across enterprise applications.
  • ✗ Vulnerability Accumulation on Virtualized Hosts: Manual patch management across hundreds of Windows and Linux virtual machines running on VMware led to delayed cycles, exposing core business infrastructure to known security vulnerabilities.
  • ✗ SLA Breaches & Unplanned Downtime: Critical background services lacked automated cluster health monitoring and structured failover controls, causing intermittent downtime that breached agreed-upon operational SLAs.
  • ✗ Lack of Standardized Audit Trails: Administrative actions, privilege escalations, and endpoint access logs were decentralised, rendering security tracking and compliance reporting slow and inefficient.

Goals

  • ✓ Centralize Enterprise Identity & Access: Implement a robust, zero-trust-aligned IAM model across Active Directory and Microsoft 365 to manage permissions seamlessly for 5,000+ endpoints and users.
  • ✓ Automate Vulnerability Remediation: Establish a predictable, zero-downtime automated OS patching lifecycle across all VMware-hosted Linux and Windows Server instances.
  • ✓ Sustain Maximum Core Service Uptime: Re-engineer core infrastructure services to guarantee high availability and consistent alignment with strict corporate SLA frameworks.
  • ✓ Enforce Least-Privilege Governance: Restrict administrative rights by introducing strict Role-Based Access Control (RBAC), multi-factor authentication (MFA), and comprehensive security logging.

Role & Responsibilities

  • ▸ Directed Hybrid Identity Management: Structured organizational units (OUs), Group Policy Objects (GPOs), and conditional access policies within Active Directory and Azure AD/M365 to enforce corporate compliance.
  • ▸ Orchestrated Enterprise Patching Pipelines: Designed and scheduled automated update ring topologies for mixed Linux (Ubuntu/RHEL) and Windows Server environments without disrupting production operations.
  • ▸ Managed Virtualization and Storage Layers: Administered core vSphere/ESXi hypervisors on VMware, provisioning resources, optimizing host density, and managing compute allocations for critical systems.
  • ▸ Hardened Operating System Baselines: Developed and deployed automated configuration scripts to continuously audit and remediate security configuration drifting on remote endpoints.
  • ▸ Monitored and Maintained SLA Compliance: Formulated granular service checks and alerting triggers to monitor critical system states, enabling immediate remediation before failures impacted end-users.

Architecture & Deployment Flow

  • → Identity Lifecycle Execution: Human Resources actions trigger automated directory provisioning, translating user roles into explicit AD groups and synchronized Microsoft 365 licensing assignments.
  • → Secure Access Challenge: When a user accesses corporate endpoints or M365 applications, Active Directory and Entra ID enforce conditional access policies, validating device compliance and triggering mandatory Multi-Factor Authentication (MFA).
  • → Virtualization Infrastructure Hosting: Enterprise business applications run within distinct Windows and Linux virtual machines orchestrated across highly available VMware ESXi clusters.
  • → Vulnerability & Telemetry Ingestion: Centralized vulnerability scanners perpetually query the internal network, generating structured reports detailing missing security patches or configuration risks across the fleet.
  • → Patch Ring Scheduling: Maintenance windows are initiated systematically using automated patching rings—deploying updates to non-critical staging clusters first before graduating to production systems.
  • → Linux and Windows Update Orchestration: Automation agents execute targeted patch deployments, safely handling service restarts, checking log health, and validating system integrity post-update.
  • → VMware Snapshot Protection: Prior to major system modifications, programmatic VMware snapshots are initiated as a cold fail-back vector, ensuring near-instantaneous recovery capability if an update destabilizes a core service.
  • → SLA Log Auditing: Infrastructure performance matrices, uptime logs, and authentication flows are aggregated continuously into centralized administration consoles for ongoing visibility and external compliance audits.

Key Implementation Details

Hybrid Identity Consolidation & RBAC

Re-engineered the corporate Active Directory structure by eliminating legacy nested groups and building an explicit, audit-ready Role-Based Access Control framework. Integrated hybrid write-back rules with Microsoft 365 to enforce uniform security policies across both on-premises hardware and cloud applications.

Zero-Downtime Patch Orchestration

Developed a dual-track rolling update framework for the VMware cluster environment. By utilizing live migration techniques (like vMotion), application workloads were transparently shifted off physical hypervisors during active patching windows, allowing host updates to occur without a single second of application downtime.

Hardened Group Policy & Baseline Enforcement

Authored and deployed comprehensive GPOs and configuration baselines that deactivated vulnerable legacy communication protocols, enforced local firewall configurations, and restricted administrative local access across 5,000+ enterprise endpoints.

Results & Impact

  • ✓ 99.99% Core Service Availability: Maintained continuous, rock-solid uptime for foundational Windows and Linux network services, fully satisfying strict corporate SLA requirements.
  • ✓ 100% Identity Visibility and Security: Successfully placed all 5,000+ enterprise users under a singular, audited identity management boundary, entirely wiping out orphaned accounts and unmapped access permissions.
  • ✓ Drastic Reduction in Exposure Windows: Compressed the mean time to patch critical system vulnerabilities from months to standard, hands-free bi-weekly cycles across the hybrid infrastructure.
  • ✓ Streamlined Security Audit Compliance: Established structured identity and event audit trails, significantly accelerating verification timelines for annual internal and external security reviews.
